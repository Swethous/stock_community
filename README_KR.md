# 📘 주식 커뮤니티 서비스 기획안 (README)

## 📌 서비스 개요
미국·일본 주식을 중심으로, 사용자들이 **실시간으로 의견을 공유**할 수 있는 커뮤니티 서비스입니다.  
주가 차트와 커뮤니티 기능을 하나의 화면에서 제공하여 **정보 탐색과 소통을 동시에** 할 수 있습니다.  
초보자도 쉽게 사용할 수 있는 **심플하고 빠른 UI**를 목표로 합니다.

---

## ❤️ 서비스에 담긴 생각 / 만들게 된 이유
제가 투자를 하면서 정보를 얻거나 종목에 대한 여론을 파악할 때는 한국의 주식 커뮤니티를 자주 이용했습니다.  
하지만 일본 투자를 시작하고 나서, 한국의 “토스 커뮤니티”처럼 **깔끔하고 가벼운 주식 커뮤니티**를 찾기 어려웠습니다.  
기존 서비스들은 UI가 복잡하거나, 커뮤니티가 메인이 아니었고 초보자가 접근하기 어려웠습니다.

그래서

> **“일본에도 초보자가 편하게 들어올 수 있는 가벼운 주식 커뮤니티를 만들자.”**

라는 생각으로 기획을 시작했습니다.

---

## 🎯 타겟 사용자

### ① 일본 주식에 관심 있는 개인 투자자 (초보 ~ 중급)
- 차트와 의견을 한 화면에서 보고 싶음  
- 실시간 반응을 빠르게 알고 싶음  

### ② 모바일 중심으로 투자 정보를 소비하는 20~40대
- 가벼운 React 기반 UI 선호  
- 빠르고 직관적인 화면 구성 필요  

### ③ 기존 커뮤니티가 불편했던 사용자
- 광고가 많고 무거운 화면에 피로감을 느끼는 사용자  
- 익명 기반으로 가볍게 의견을 나눌 수 있는 공간을 찾는 사람  

---

## 💡 서비스 이용 이미지
- 홈 화면에서 **인기 / 급상승 / 급락** 종목을 빠르게 탐색  
- 종목 상세 화면에서 **실시간 차트 + 거래량 + 커뮤니티** 동시 확인  
- 관심 종목을 **북마크**하여 즉시 확인  
- 댓글로 의견을 공유하며 투자 판단에 도움  

---

## 🧲 사용자 유입 전략
- Twitter(X) 중심의 투자자 커뮤니티 홍보  
- 인기 종목 자동 랭킹 기능을 통한 자연 유입  
- 초보자를 위한 직관적 UI 제공  
- SEO 최적화 : “종목명 + 커뮤니티/토론” 키워드 검색 노출  
- 모바일 웹 최적화로 접근성 강화  

---

## 🚀 차별화 포인트 & 강점

### ① 초경량 · 초고속 UI
React + Lightweight Charts + 캐싱 구조로  
기존 서비스 대비 **매우 빠르게 차트와 데이터를 로딩**할 수 있습니다.

### ② 최소 UI 구성으로 집중도 상승
필요한 핵심 기능만 제공:  
- 간단 차트  
- 커뮤니티(댓글)  
- 관심종목  

군더더기 없는 화면으로 사용자 경험을 극대화합니다.

## 🧩 기능 후보

### ✔ MVP (최소 기능 출시 목표)
- 회원가입 / 로그인 (JWT 기반)  
- 주가 차트 표시 (Yahoo Finance API)  
- 인기 / 급상승 / 급락 종목 리스트  
- 종목 상세 화면  
  - 차트  
  - 거래량 그래프  
  - 댓글 (작성 / 조회)  
- 관심종목(북마크) 기능  

---

### ✔ 본 출시 버전

## 🛠️ 기술 스택

### 🔹 백엔드 (API Server)
- Ruby on Rails 7 (API 모드)  
- JWT 인증  
- rack-cors  
- PostgreSQL  
- Redis (캐싱, 세션, 백그라운드 작업)  
- Sidekiq (정기적 데이터 업데이트)  
- Yahoo Finance API  

### 🔹 프론트엔드
- React 18  
- Vite  
- Zustand  
- axios  
- lightweight-charts (npm)  
- ESLint / Prettier  

### 🔹 배포 환경
- 프론트엔드 : Vercel  
- 백엔드 : Fly.io (Docker)  
- DB : Supabase PostgreSQL  
- 이미지 저장소 : Supabase Storage
